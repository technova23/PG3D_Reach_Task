## P04 CoppeliaSim 4.9 compatibility

Observed on the workstation with `COPPELIASIM_ROOT` pointing at
`CoppeliaSim_Edu_V4_9_0_rev6_Ubuntu22_04`:

- CoppeliaSim launched and reported `ModuleNotFoundError: No module named 'zmq'` from its
  `python/pythonLauncher.py`;
- `PyRep.launch(scene_file=...)` left the default scene loaded, leading to
  `RuntimeError: Handle Panda does not exist`;
- direct PyRep joint reads corrupted memory because PyRep's CFFI wrappers used `float*` against
  CoppeliaSim 4.9 double-precision legacy API symbols;
- RLBench reset validation called dropped path/IK APIs (`simGetConfigForTipPose`,
  `simGenerateIkPath`), which are only needed for waypoint/demo feasibility checks;
- first-frame force fields could report `No value available yet`;
- writing the MP4 while CoppeliaSim was still alive could trip a CoppeliaSim shutdown segfault.

Fix:

- kept CoppeliaSim 4.9.0 rev6 as a supported M1 observation path instead of forcing a 4.1.0 install;
- added `pyzmq` and `cbor2` to the `rlbench` extra and pointed CoppeliaSim's Python add-on launcher
  at the active pg3d virtualenv;
- patched PyRep launch to load RLBench `task_design.ttt` with `simLoadScene` after startup;
- routed precision-sensitive PyRep wrappers through CoppeliaSim `*_D` symbols;
- skipped RLBench waypoint validation only for observation-only smoke/save paths;
- disabled optional force/touch-force fields for the first saved observation;
- moved NPZ/JSON/MP4 artifact writing after simulator shutdown;
- recorded the compatibility decision in `docs/adr/0007-coppeliasim-49-rlbench-compat.md`.

Validation:
