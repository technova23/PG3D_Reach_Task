# Prompt R — Update docs and worklog

Goal:
Update project docs after the current coding session.

Read first:
- `AGENTS.md`
- `docs/project_proposal.html` (DO NOT SKIP. READ IN FULL DETAIL; this is critical to the project.)
- `docs/status.md`
- recent git diff

Tasks:
1. Summarize what changed in `docs/status.md`.
2. Add or update today's worklog in `docs/worklog/YYYY-MM-DD.md` with:
   - objective,
   - files changed,
   - commands run and outcomes,
   - blockers/risks,
   - next recommended task.
3. Update `docs/runbooks/commands.md` if any command changed.
4. Propose an ADR if a durable design decision was made.
5. Do not invent results that were not run.

Done when:
- Docs reflect the actual current repo state.
